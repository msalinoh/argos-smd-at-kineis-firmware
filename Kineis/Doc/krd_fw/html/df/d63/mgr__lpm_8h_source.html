11 /**
12  * @page mgr_lpm_page Low Power Mode Manager (generic code)
13  *
14  * This page is presenting the low power mode manager. This software entity is in charge to enter
15  * and exit low power modes thanks to client requests.
16  *
17  * @section mgr_lpm_design Design
18  *
19  * Here is the rough design we want to apply on low power mode manager. It has to be detailled
20  * deeper.
21  *
22  * The LPM manager should work as a client/server model.
23  * When enterLPM() API is called, the LPM manager:
24  * * Asks the deepest LPM allowed by each client
25  * * The shallowest mode is chosen
26  * * Each client is notified by the manager
27  * * Finally, the manager sets the chosen LPM
28  *
29  * enterLPM()
30  * \_________
31  * | MGR_LPM |
32  * |_________|
33  * // || \\
34  * // || \\
35  * Client1 Client2 ClientX
36  *
37  * Refer to mgr_lpm.h for extra details on the Apis
38  */

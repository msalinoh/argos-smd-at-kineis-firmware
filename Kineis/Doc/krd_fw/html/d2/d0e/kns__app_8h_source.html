8 /**
9  * @page kns_app_page APP task
10  *
11  * This page is presenting Two APPlication task example which exercices the Kineis stack SW
12  * (\ref KNS_APP) in a way to send user data to the Kineis network..
13  *
14  * Two applications tasks are available:
15  * * A standalone app (\ref KNS_APP_stdln_loop) is sending one user message (4 retransmit each 60s)
16  * * A GUI app (\ref KNS_APP_gui_loop) is dedicated to communicate with the Kinéis Device Interface
17  * (KDI) through AT commands (\ref mgr_at_cmd_page) over an UART link (9600 bauds, 8, N, 1).
18  *
19  * Those application needs some configurations from user such as:
20  * * Credentials (ID, ADDR, Secret key)
21  * * Radio configuration (UL frequency and modulation)
22  * * Kineis protocol configuration (a.k.a. MAC profile)
23  * Refer to "getting started" section for details.
24  *
25  * In case of GUI APP, the format of an AT command is like:
26  * * "<pattern>=?\r\n" (e.g. "AT+PING=?\r\n") to get information
27  * * "<pattern>=<params>\r\n" (e.g. "AT+TX=<user_data>\r\n") to set information
28  * The list of supported AT commands is available through \ref cas_atcmd_list_array.
29  *
30  * Communication between tasks (APP and MAC) in done through events posted in OS Queues.
31  * By the end, those applications shows how to:
32  * * Invoke the kineis stack to send USER DATA
33  * * Wait notification from the Kineis stack once the transmission is completed
34  *
35  * @section kns_app_subpages Sub-pages
36  *
37  * * @subpage mgr_at_cmd_page
38  */

65 /** @brief This fct is a very basic standalone application
66  *
67  * This application is posting some send-data event to MAC layer of Kineis software stack in loop as
68  * long as there is no error.
69  *
70  * Depending on the defined MAC protocol:
71  * * BASIC: APP is trying to send 2 data at start but kineis stack can only accept the first one.
72  * This data is sent once only, then is goes into lowpower mode.
73  * * BLIND: APP is trying to send 5 data at start but 4 are accepted. Each data is re-transmitted
74  * (retx) several times periodically as per the configuration of the protocol. Once a data is fully
75  * transmitted, APP will try to send a new data again.
76  */
77 void ‪KNS_APP_stdln_loop(void);

18 /**
19  * @page kns_mac_page MAC task (Kineis Stack entry point)
20  *
21  * This page is presenting the task holding the Kineis stack (\ref KNS_MAC) and how to use it
22  *
23  * The \ref KNS_MAC_task can be started by the OS as a classical task.
24  * Once started, the task will internally go through each queue waiting for any new event coming
25  * from the outside world (APP task, KNS_RF events, INFRA interrupts).
26  *
27  * The first thing to really start the kineis stack is to init the profile with \ref KNS_MAC_init
28  * direct API.
29  *
30  * Once initialized, the stack is able to receive envents from the external worlds (APP task,
31  * KNS_RF, INFRA ISR).
32  *
33  * * Top APIs APP-to-MAC : cf events in \ref KNS_MAC_appEvtId_t
34  * * Top APIs MAC-to-APP : cf events in \ref KNS_MAC_srvcEvtId_t
35  * * Bottom APIs MAC-to-KNS_RF: cf direct APIs \ref kns_rf.h
36  * * Bottom APIs KNS_RF-to-MAC: cf events in \ref KNS_RF_eventId_t reported to MAC through callbacks
37  *
38  * @section kns_app_subpages Sub-pages
39  *
40  * * @subpage mcu_aes_page
41  * * @subpage mcu_nvm_page
42  * * @subpage mcu_tim_page
43  * * @subpage kns_rf_page
44  */
